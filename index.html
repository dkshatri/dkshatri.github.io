Machine Translations

Role: Product Manager

Timeline: 12 months
+
Case Study

As the Product Manager and Strategy Lead, I led the Machine Translations initiative to cut cost and improve speed at global scale. The CFO sponsored program delivered a unified portal and translation pipeline with a translation memory layer to maximize accuracy and reuse.

Leadership & Strategy CFO Sponsored Accuracy Focus Agile Delivery Role: Product Manager
See outcomes → Explore solution
🌍

Portal for on-demand requests and a bulk pipeline with translation memory reuse

Problem

  • ✅Rising cost from manual processes across departments
  • ✅Slow delivery blocked global launches
  • ✅Inconsistent quality and fragmented tools
  • ✅No central pipeline for requests

Goals

  • ✅Create a portal for new content
  • ✅Build a bulk pipeline for existing content
  • ✅Implement translation memory for reuse
  • ✅Improve accuracy with language-specific engines

Approach

DISCOVER

Interviewed content owners, mapped flows, audited glossary use.

DECIDE

Chose AWS Translate with custom terms, defined QA gates and reuse rules.

DELIVER

Automated intake → translation → review → publish with dashboards.

Solution overview

Unified portal for request intake plus a phased pipeline for bulk translation. Translation memory captures prior sentences for reuse. Accuracy is improved by selecting the best engine per language across AWS and Google.

🧩
Language engine match

Choose the most accurate engine per language for better output

👥
Stakeholder alignment

Requirements sorted into must-have, should-have, could-have, and won't-have with reasoning

🛠️
Phased delivery

Iterative releases with feedback to drive adoption

📄
Translation memory

Reuse existing sentences for consistency and savings

Highlights
  • ✅CFO recommended initiative
  • ✅One portal and one pipeline for the enterprise
  • ✅Measured and verified savings and speed
  • ✅Accuracy validated with side-by-side checks

What I Owned

  • ✅Product strategy, roadmap, and OKRs
  • ✅Workflow design, QA rules, and dashboards
  • ✅Vendor evaluation and rollout playbook
  • ✅End-to-end delivery oversight

Technical challenges & workarounds

Addressed platform limits and accuracy risks with preprocessing, data design, and guardrails.

AWS service limits
  • ✅Single custom terms file allowed
  • ✅Document and per-cell size limits
  • ✅No edit option for uploaded terms
  • ✅No empty cells allowed in custom terms or parallel data

Workarounds include splitting and merging term files, validating cells, and staging uploads to avoid loss.

Accuracy and quality

Matched engines by language and verified outputs with sampling and review. Translation memory reinforced phrasing over time.

Shared pipeline governance

Consolidated requirements and prioritized must-have items. Created a clear intake and SLA model across departments.

Machine Translations Mobile Integration

Role: Product Manager

Timeline: 4 months
+
Case Study

As the Product Manager, I led the integration of machine translated content into the mobile app while maintaining a seamless user experience. This required UX design to support a module specific language switch alongside the global app level switch.

Mobile UX Localization User Research Role: Product Manager
See outcomes → Explore solution
📱

Translated module content on mobile with a clear temporary language switch

Problem

  • ✅Existing global language switch managed all localized content
  • ✅New translations applied to only one module
  • ✅Two different switches would confuse users and degrade UX

Goals

  • ✅Integrate machine translated content seamlessly on mobile
  • ✅Maintain clarity and consistency between global and module level switches
  • ✅Gather and apply feedback through user testing

Approach

DISCOVER

Mapped current UX flows and ran brainstorming sessions

DECIDE

Proposed a module specific switch with a clear note on scope

DELIVER

Tested with focus groups, iterated, and shipped with positive feedback

Solution overview

A dedicated temporary language switch lives inside the target module. A note clarifies scope and that leaving the module resets the app to the global language, preserving the overall experience.

🧩
Seamless integration

Global switch remains authoritative beyond the module

👥
User centered

Feedback loops ensured clarity and confidence

🛠️
UX consistency

No surprises when navigating in or out of the module

📄
Positive feedback

Focus groups favored the clear scoped control

Outcomes

  • ✅Translated content launched in the target module
  • ✅Consistent user experience across the app
  • ✅Focus groups confirmed clarity and ease of use
